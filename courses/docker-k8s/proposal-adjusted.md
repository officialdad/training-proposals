# DevOps Fundamentals with Docker & Introduction to Kubernetes

**TRAINING PROPOSAL** — Crux Institute

- **Duration:** 2 Days · 16 Hours
- **Classification:** Digital Skills
- **Prepared for Review:** April 2026

---

## 01 Course Overview

This programme introduces participants to the fundamentals of DevOps and containerisation, with a strong focus on Docker and supporting tools such as Git and GitHub. Participants will learn how modern software teams collaborate, automate, and deploy applications using industry-standard practices.

The programme is designed to be realistic and achievable within 2 days, prioritising depth over breadth. Approximately 80% of the content focuses on DevOps fundamentals and Docker, 10% covers Git/GitHub workflows for CI/CD, and the final 10% provides a crash-course preview of Kubernetes to prepare participants for a follow-up intermediate session.

Aligned with Industry 4.0 and cloud-native transformation, this programme equips participants with the foundational skills required to begin their DevOps journey.

## 04 Course Objectives

At the end of this programme, participants will be able to:

1. Understand core DevOps principles and the software delivery lifecycle
2. Use Git and GitHub for version control and team collaboration
3. Build, run, and manage Docker containers confidently
4. Create and optimise Docker images using Dockerfiles
5. Orchestrate multi-container applications with Docker Compose
6. Understand the basics of CI/CD pipelines and container registries (GHCR)
7. Gain a foundational awareness of Kubernetes and container orchestration

## 05 Learning Outcomes

Upon completion of the programme, participants will be able to:

- Explain DevOps principles and the role of containerisation
- Apply Git and GitHub workflows in a team environment
- Build, run, and debug containerised applications using Docker
- Write Dockerfiles and docker-compose files for real applications
- Understand the building blocks of a CI/CD pipeline
- Publish and pull Docker images using GitHub Container Registry (GHCR)
- Describe what Kubernetes is and when it is needed

## 07 Course Outline

*2-day programme · 16 hours of intensive, hands-on learning*

### Day 1: DevOps Fundamentals & Docker Essentials (8 Hours)

**Hour 1 — Introduction to DevOps**
- What is DevOps and why it matters
- The software delivery lifecycle
- Challenges in traditional deployment
- DevOps culture and principles

**Hour 2 — Version Control with Git**
- Introduction to Git and version control
- Core Git commands (clone, commit, push, pull)
- Branching and merging fundamentals
- Resolving conflicts

**Hour 3 — Collaboration with GitHub**
- Working with remote repositories
- Pull requests and code review workflows
- GitHub Issues and project collaboration
- Introduction to GitHub Actions

**Hour 4 — Introduction to Containerisation & Docker**
- Why containers? Problems they solve
- Virtual machines vs containers
- Docker architecture and components
- Installing and verifying Docker

**Hour 5 — Working with Docker Containers**
- Docker CLI fundamentals
- Running and managing containers
- Working with Docker Hub and public images
- Inspecting and debugging containers

**Hour 6 — Building Docker Images**
- Writing your first Dockerfile
- Building and tagging custom images
- Understanding image layers
- Managing local images

**Hour 7 — Docker Networking & Volumes**
- Container networking basics
- Port mapping and exposure
- Volumes and data persistence
- Environment variables and configuration

**Hour 8 — Hands-On Lab: Git + Docker Basics**
- Cloning a sample project from GitHub
- Dockerising and running the application
- Committing and pushing changes
- Review and discussion

### Day 2: Docker in Practice, CI/CD & Kubernetes Preview (8 Hours)

**Hour 1 — Dockerfile Deep Dive & Optimisation**
- Review of Day 1 concepts
- Multi-stage builds
- Reducing image size
- Caching and build efficiency

**Hour 2 — Multi-Container Apps with Docker Compose**
- Introduction to Docker Compose
- Writing docker-compose.yml files
- Managing services and dependencies
- Running a multi-container application

**Hour 3 — Docker Best Practices**
- Security considerations
- Managing image versions and tags
- Resource management
- Common pitfalls to avoid

**Hour 4 — Hands-On Lab: Multi-Container Application**
- Building a web app with a database using Compose
- Managing logs and debugging
- Stopping, restarting, and cleaning up
- Review and discussion

**Hour 5 — Introduction to CI/CD & Container Registries**
- What is CI/CD and why it matters
- The role of Docker in CI/CD pipelines
- Introduction to container registries (Docker Hub, GitHub Container Registry)
- Walkthrough of a simple GitHub Actions workflow

**Hour 6 — Hands-On Lab: CI/CD Pipeline with GHCR**
- Dockerising a sample application
- Pushing code to GitHub
- Automating image builds via GitHub Actions
- Publishing the image to GitHub Container Registry (ghcr.io)
- Pulling and running the published image

**Hour 7 — Kubernetes Crash Course (Preview)**
- What Kubernetes is and why it exists
- High-level architecture (nodes, pods, clusters)
- Key concepts: deployments and services
- Live demo: deploying a container to Minikube

**Hour 8 — Hands-On Lab: Minikube & Wrap Up**
- Follow along for Minikube setup
- Recap of key concepts
- Final Q&A and discussion
- Feedback and closing

## 01 Course Overview

Schools run on repetitive workflows — attendance logging, permission slips, leave approvals, parent communications, event sign-ups — and most of these still depend on manual effort or on tools that sit outside the organisation's core subscription. Many teams reach for third-party automation platforms or build everything inside Google AppSheet, only to discover that AppSheet's app-rendering layer adds performance overhead that a pure workflow rarely needs.

This 1-day programme keeps every automation inside the Google Workspace subscription ISKL already pays for, and anchors on the newest and most capable native tool: **Google Workspace Studio** (generally available since December 2025). With Workspace Studio, staff describe what they want in plain language and Gemini builds the flow — agents that read Gmail, write to Sheets, update Calendar, and access Drive without API keys, connectors, or external hosting. It is, in effect, a native no-code equivalent to platforms like n8n, with nothing to install and nothing to maintain.

Around that anchor, participants use **Gemini and custom Gems** as an automation copilot, **Google Forms and Sheets** as the data-capture and trigger backbone, and a single **copy-and-run Apps Script recipe** for the cases no-code does not yet cover. The day closes by treating tool selection as a skill in its own right — so staff leave able to decide when a workflow belongs in Workspace Studio, when a lightweight script is the answer, and when AppSheet is genuinely warranted, ending the performance frustrations of forcing every problem into one tool.

## 04 Course Objectives

At the end of this programme, participants will be able to:

1. Compare Google's native automation tools — Workspace Studio, Apps Script, and AppSheet — and select the right one for a given school workflow
2. Use Gemini and a custom Gem to accelerate a recurring communication or content task
3. Configure Google Forms and Sheets as the data-capture and trigger backbone for automated workflows
4. Build, test, and schedule a multi-step Workspace Studio automation that connects Gmail, Sheets, Calendar, and Drive
5. Run and adapt a copy-and-run Apps Script recipe (mail merge) without scripting from scratch
6. Design and ship an end-to-end ISKL workflow — without any third-party tools or external hosting

## 05 Learning Outcomes

Upon completion of the programme, participants will be able to:

- Articulate the trade-offs between Workspace Studio, Apps Script, and AppSheet, and migrate a slow AppSheet workflow to a lighter native flow
- Use Gemini's side panel and a custom Gem to accelerate everyday administrative work
- Turn a Google Form submission into an automated, multi-step workflow across the Workspace suite
- Build, test, schedule, and share a Workspace Studio agent independently
- Run and adapt a pre-built Apps Script recipe without writing scripts from scratch
- Apply basic governance and data-privacy practice when automating workflows that touch student or staff data

## 07 Course Outline

*1-day programme · 8 hours of intensive, hands-on learning*

**Hour 1 — The Google-Native Automation Landscape**
- Why stay native: keeping workflows inside the existing Workspace subscription, no third-party hosting
- The three native tools: Workspace Studio (no-code flows), Apps Script (low-code recipes), AppSheet (no-code apps)
- Choosing the right tool: why a pure workflow does not need AppSheet's app-rendering overhead
- Mapping common school workflows to the right tool

**Hour 2 — Gemini in Workspace & Custom Gems**
- Gemini in the side panel across Gmail, Docs, Sheets, and Drive
- "Help me write / organise / analyse" in the flow of daily work
- Building a custom Gem for a recurring task (e.g. drafting parent notices in a house style)
- Responsible AI use: reviewing AI output before it goes out

**Hour 3 — Forms & Sheets: The Automation Backbone**
- Designing a Form that captures clean, structured data
- The Form → Sheet pipeline and where automations hook in
- Structuring a Sheet so downstream automation stays reliable
- Identifying trigger points for an automated workflow

**Hour 4 — Workspace Studio I: Your First Agent**
- Anatomy of a Workspace Studio agent: trigger, steps, actions
- Building a first agent entirely in plain language
- Connecting Gmail, Sheets, Calendar, and Drive without API keys
- Running, inspecting, and debugging a flow

**Hour 5 — Workspace Studio II: Multi-Step Flows**
- Chaining steps: read Gmail → update a Sheet → notify a person
- Adding conditions and branches in plain language
- Using Gemini inside a flow to summarise or classify content
- Common patterns: notifications, logging, routing

**Hour 6 — Workspace Studio III: Scheduling, Governance & Migrating Off Slow AppSheet**
- Event triggers vs scheduled runs, and sharing agents with a team
- Understanding execution limits by plan and designing within them
- Governance and data privacy: who can build, who can run, where data flows
- Migrating a notification / approval workflow from a slow AppSheet app to Workspace Studio

**Hour 7 — Apps Script Recipes: When No-Code Runs Out**
- What Apps Script is and the cases where it beats no-code
- The official mail-merge recipe: Sheets (data) + Gmail (send) + Apps Script (engine)
- Copy, authorise, and run a recipe safely — no scripting from scratch
- Adapting a recipe's variables to a school use case and knowing when to hand it to IT

**Hour 8 — Hands-On Lab: Ship a Forms-Triggered ISKL Workflow**
- Build end-to-end: Form submission → Workspace Studio flow → approver notification → logged decision
- Reskin the approval pattern to a real ISKL case (leave, permission slip, purchase, or event sign-up)
- Test with sample submissions and verify the audit trail in Sheets
- Present the workflow and the tool-selection rationale; trainer observation and feedback

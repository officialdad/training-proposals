# Claude Cowork: AI Agent Automation for Everyday Knowledge Work

**TRAINING PROPOSAL** — Trainocate / ISKL

- **Duration:** 1 Day · 8 Hours
- **Classification:** Digital Skills
- **Prepared for Review:** June 2026

---

## 01 Course Overview

Schools lose enormous staff time to work that is tedious but not technically complex: sorting files, compiling reports from scattered sources, drafting routine communications, and pulling data out of PDFs, forms, and receipts. Traditional automation asks staff to *build* the solution first — design a flow, configure an app, learn a tool — which is exactly the barrier most administrative and operations staff do not have time to clear.

Claude Cowork takes a different approach. It is Anthropic's agentic desktop "coworker" — generally available in 2026 and running on the Macs ISKL staff already use. Rather than building a flow, staff describe the task in plain language; Claude produces a step-by-step plan, waits for approval, and then executes it — reading and writing local files, using the browser and everyday applications, and connecting to tools such as Google Drive and Gmail so it works directly with the school's existing Google data. Scheduled tasks let it run recurring work — a morning email digest, a weekly report — automatically.

This 1-day programme is built for non-technical school staff. It is anchored on Cowork's permission-first design: every consequential action is reviewed before it runs, and access is scoped to only the folders and tools a task needs. Participants learn to delegate real ISKL work safely, with particular attention to the privacy of student and staff data, and leave able to automate multi-step tasks without building a single flow or writing a line of code.

## 03 Entry Requirements

- A macOS computer (Claude Cowork runs in the Claude desktop app)
- A paid Claude subscription (Pro, Max, Team, or Enterprise) with Cowork enabled
- Day-to-day familiarity with files, folders, and Google Workspace (Drive, Gmail)
- No programming experience required
- **Pre-engagement:** ISKL to provision Claude licences (Team or Enterprise recommended for a staff rollout and admin controls) and confirm its data-handling policy for use with an AI agent

## 04 Course Objectives

At the end of this programme, participants will be able to:

1. Explain how Claude Cowork's agentic, plan-then-execute model differs from flow-builder and app-builder automation
2. Set up Claude Cowork on macOS, grant scoped file and folder access, and connect it to Google Drive and Gmail
3. Delegate a multi-step task in plain language and review, adjust, or approve Claude's plan before it executes
4. Automate document preparation and the extraction of structured data from unstructured files such as PDFs and reports
5. Configure a scheduled task that runs recurring work automatically
6. Apply permission-first and data-privacy practices appropriate to handling student and staff information

## 05 Learning Outcomes

Upon completion of the programme, participants will be able to:

- Decide when to delegate a task to an AI agent versus build a traditional flow
- Install and configure Claude Cowork, scope its access safely, and connect it to their Google data
- Write a clear task brief, steer Claude's plan, and supervise execution to completion
- Produce finished deliverables — sorted files, structured drafts, reports, and extracted datasets — with appropriate quality checks
- Set up and monitor a recurring scheduled automation
- Handle student and staff data responsibly when working with an autonomous agent

## 07 Course Outline

*1-day programme · 8 hours of intensive, hands-on learning*

**Hour 1 — A New Kind of Automation: Meet Claude Cowork**
- Three paradigms: flow builders, app builders, and agentic coworkers
- How Cowork works: describe → plan → approve → execute
- Where it runs: the desktop, local files, your apps, and the browser
- When to delegate to an agent versus build a flow

**Hour 2 — Setup, Permissions & Connecting Your Google Data**
- Installing the Claude desktop app and signing in (plan requirements)
- Granting scoped folder access — the permission-first model
- Connecting Google Drive and Gmail
- Safety defaults: reviewing plans, approving actions, stopping a run

**Hour 3 — Your First Delegated Task**
- Writing a clear task brief in plain language
- Reading Claude's step-by-step plan and steering it
- Approving, adjusting, or cancelling before anything happens
- Inspecting results and iterating

**Hour 4 — File & Inbox Automation**
- Sorting, renaming, and deduplicating a messy folder
- Triaging and summarising an inbox
- Filing attachments to the right Drive location
- Verifying before anything is moved or sent

**Hour 5 — Document Preparation & Data Extraction**
- Turning a pile of source files into a structured draft
- Extracting data from PDFs, forms, and reports into a spreadsheet
- Generating a report or presentation deliverable
- Quality and accuracy checks on agent output

**Hour 6 — Scheduled & Recurring Automations**
- Scheduled tasks: setting a cadence
- A worked example: a morning digest or weekly report
- Long-running tasks and parallel sub-agent work
- Monitoring scheduled runs and handling failures

**Hour 7 — Governance, Privacy & Safe Delegation for Schools**
- Handling student and staff data with an autonomous agent
- Scoping access narrowly: what Claude should and should not touch
- Reviewing consequential actions; accountability and audit
- Organisational rollout: Team and Enterprise controls and policy

**Hour 8 — Hands-On Lab: Automate a Real ISKL Task End-to-End**
- Choose a real task: report compilation, communications drafting, data extraction, or file organisation
- Brief Claude, review the plan, and execute with oversight
- Add a scheduled run where it fits the workflow
- Present the automation and the privacy/oversight decisions made; trainer observation and feedback

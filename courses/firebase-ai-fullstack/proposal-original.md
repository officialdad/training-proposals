## 01 Course Overview

Many IT teams in schools and small organisations grow up inside Google AppScript — automating spreadsheets, sending emails, and stitching workflows together without ever leaving the Google ecosystem. AppScript is productive at small scale, but it hits hard ceilings the moment a project needs proper hosting, multi-user authentication, a real database, or version-controlled collaboration across a team.

This 1-day programme is built for IT teams making exactly that transition. Participants will leave AppScript behind for a single day and ship a complete fullstack web application on Firebase — Google's unified platform for authentication, databases, and hosting — using vanilla HTML and JavaScript that maps naturally to skills they already have.

The programme treats two modern practices as non-negotiable: proper version control with Git and GitHub from the first commit, and disciplined use of AI-assisted coding agents through Antigravity IDE. Participants will not just write code with AI assistance; they will learn to review, accept, and reject AI suggestions before each commit lands, building habits that scale to professional teams.

## 04 Course Objectives

At the end of this programme, participants will be able to:

1. Explain the architectural shift from AppScript-bound spreadsheets to a standalone Firebase fullstack web application
2. Configure a Git repository with GitHub remotes, branch workflow, and commit hygiene appropriate for team collaboration
3. Set up Antigravity IDE with AI-assisted coding agents and integrate Firebase tooling into the development workflow
4. Provision a Firebase project with Authentication, Firestore, and Hosting enabled
5. Build a frontend that reads from and writes to Firestore using the Firebase Web SDK
6. Deploy a complete fullstack web application to Firebase Hosting via the Firebase CLI
7. Use AI coding agents responsibly to scaffold code, debug errors, and review changes before each commit

## 05 Learning Outcomes

Upon completion of the programme, participants will be able to:

- Articulate the limits of AppScript and the benefits of a Firebase fullstack architecture for school and small-team IT
- Operate core Git and GitHub workflows: `init`, `clone`, branch, pull request, merge, and resolving simple conflicts
- Use Antigravity IDE's AI agents to write, refactor, and explain code with appropriate human review
- Configure Firebase Authentication, Firestore security rules, and Hosting from the Firebase console
- Read, write, and query Firestore documents from a browser frontend using the Firebase Web SDK
- Deploy a live web application and roll back to a previous version via the Firebase CLI and console

## 07 Course Outline

*1-day programme · 8 hours of intensive, hands-on learning*

**Hour 1 — From AppScript to Modern Web: Why Firebase**
- Limits of AppScript-bound apps: scaling, hosting, multi-user access, version control
- Anatomy of a fullstack web application: frontend, database, hosting, authentication
- Firebase as a unified platform: Auth, Firestore, and Hosting from one console
- Mapping AppScript habits to modern web equivalents

**Hour 2 — Git & GitHub Foundations for IT Teams**
- Why version control: the AppScript editor problem solved with Git history
- Initialising a repository, staging, committing, and writing meaningful commit messages
- Remote workflow: pushing to GitHub, working with branches and pull requests
- Reviewing diffs and resolving simple merge conflicts

**Hour 3 — Antigravity IDE & AI-Assisted Development**
- Installing Antigravity IDE and signing in
- Project setup: opening a folder and connecting the Git repository
- AI agents in practice: prompting for scaffolding, asking for explanations, accepting and rejecting suggestions
- Responsible AI use: reviewing every line before commit and avoiding silent agent edits

**Hour 4 — Firebase Project Setup & Authentication**
- Creating a Firebase project from the console
- Enabling Email/Password and Google sign-in providers
- Installing the Firebase CLI and linking the local project (`firebase init`)
- Adding the Firebase SDK to a vanilla HTML/JS page and wiring sign-in

**Hour 5 — Firestore: The Database Layer**
- Firestore data model: collections, documents, fields
- Writing the first document from the browser console
- Querying collections with `where`, `orderBy`, and `limit`
- Security rules basics: restricting writes to authenticated users only

**Hour 6 — Frontend: HTML/JS App Talking to Firestore**
- Building a simple CRUD page: list, create, update, delete documents
- Listening to real-time updates with `onSnapshot`
- Handling errors and loading states in the UI
- Using AI agents in Antigravity to refactor a working component

**Hour 7 — Deploy with Firebase Hosting + GitHub Versioning**
- Configuring `firebase.json` for static hosting
- Running `firebase deploy` to a live `*.web.app` URL
- Pushing the final code to GitHub and tagging the release
- Rolling back a deployment via the Firebase console

**Hour 8 — Hands-On Lab: Ship a Complete Firebase Fullstack App**
- Build end-to-end: Auth + Firestore CRUD + Hosting, scaffolded with AI assistance
- Commit history must show clean Git progression (no single dump commit)
- Deploy to a live URL and verify sign-in, data persistence, and security rules
- Simulate a regression, roll back, and document the fix in a pull request
